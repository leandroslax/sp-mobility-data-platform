# Release v1.0.0

Primeira release estável do `sp-mobility-data-platform`, consolidando o projeto como um case profissional de engenharia de dados em Azure + Databricks.

## Highlights

- pipeline Databricks validado ponta a ponta
- CI em verde com validações de sintaxe, JSON, YAML, lint, testes e Terraform
- CD em `develop` e `main` com deploy validado para Databricks
- governança, qualidade e observabilidade incorporadas ao fluxo
- promoção concluída até `main` com deploy aprovado em `prod`

## O que entrou nesta release

- estabilização dos notebooks Databricks com remoção de dependências frágeis de `%run`
- ajuste de paths de notebooks para o formato real do Workspace Databricks
- alinhamento de secrets e nomes de chaves com o ambiente validado
- fallback local para GTFS quando a origem remota retorna `403 Forbidden`
- correção do `quality_runner` e dos jobs para execução com os paths corretos
- migração do job para o cluster existente `sp-mobility`
- validação manual do pipeline completo: setup, governança, ingestão, bronze, silver, gold e quality
- documentação fortalecida em `README`, `CONTRIBUTING`, Terraform, arquitetura e decisões
- criação de base inicial de testes unitários e de integração
- evolução do CI/CD para deploy e verificação de notebooks e jobs no Databricks
- configuração de GitHub Environments `dev` e `prod`
- proteção das branches principais e release promovida até `main`
